# anyOf

Creates an any of validation action.

```ts
const Action = v.anyOf<TOptions, TMessage, TInput>(options, message);
```

## Generics

- `TOptions` <Property {...properties.TOptions} />
- `TMessage` <Property {...properties.TMessage} />
- `TInput` <Property {...properties.TInput} />

## Parameters

- `options` <Property {...properties.options} />
- `message` <Property {...properties.message} />

### Explanation

With `anyOf` you can combine alternative constraints for a value that was already typed earlier in a `pipe`. The input is accepted if at least one option succeeds. If all options fail, `anyOf` returns one top-level issue with the option issues as subissues.

The `options` array must contain at least two actions. `anyOf` only supports sync validation actions and the sync `guard` action. It does not select transformed outputs or adopt changed option values, so async actions, metadata actions, always-success transformations, and type-changing or value-changing transformations are intentionally not supported.

Use <Link href="../union/">`union`</Link> when alternatives have different input data types or different transformations. Use <Link href="../variant/">`variant`</Link> for discriminated object unions.

## Returns

- `Action` <Property {...properties.Action} />

## Examples

The following examples show how `anyOf` can be used.

### Website schema

Schema to validate a domain or URL.

```ts
const WebsiteSchema = v.pipe(
  v.string(),
  v.anyOf([v.domain(), v.url()], 'Please enter a domain or URL.')
);
```

### CSS size schema

Schema to validate a CSS size with a narrowing output type.

```ts
const CssSizeSchema = v.pipe(
  v.string(),
  v.anyOf([
    v.guard((input): input is `${number}px` => /^\d+px$/.test(input)),
    v.guard((input): input is `${number}rem` => /^\d+rem$/.test(input)),
  ])
);
```

### Transformed alternatives

Use `union` when alternatives transform values differently.

```ts
const NumberSchema = v.union([
  v.number(),
  v.pipe(v.string(), v.toNumber(), v.integer()),
]);
```
